✦ 실제 Supabase와 Polar.sh의 API 키를 사용하여 테스트하려면 실제 외부 서비스와 통신할 수 있는 환경을
  구성해야 합니다. 아래의 순서에 따라 진행하시면 실데이터 기반의 엔드투엔드(E2E) 테스트가 가능합니다.

  ---

  1. API 키 준비 및 .env 설정

  먼저 각 서비스의 관리자 페이지에서 필요한 키를 가져와 backend/.env 파일에 입력합니다.

   * Supabase (Project Settings > API):
       * SUPABASE_URL: 프로젝트 URL
       * SUPABASE_ANON_KEY: Anon Public Key
       * SUPABASE_JWT_SECRET: JWT Secret (JWT 검증에 필수)
   * Polar.sh (Settings > Developers):
       * POLAR_API_KEY: Personal Access Token 발급
       * POLAR_ORGANIZATION_ID: 조직 ID (Organization Settings에서 확인)
       * POLAR_WEBHOOK_SECRET: 웹훅 엔드포인트 생성 시 발급되는 Secret

  2. 서버 실행

  터미널에서 실제 서버를 구동합니다.

   1 cd backend
   2 # 1. 환경변수 로드 확인
   3 # 2. 서버 가동
  시나리오 A: 인증 및 유저 동기화 (/auth/me)
   1. JWT 획득: Supabase 대시보드의 Authentication > Users에서 테스트 유저를 만들거나,
      프론트엔드/Postman을 통해 로그인하여 access_token을 얻습니다.
   2. API 호출:
       * Header: Authorization: Bearer <SUPABASE_JWT>
       * Header: x-project-id: your-app-id
       * 결과: 우리 DB의 users 테이블에 유저가 자동으로 생성되는지 확인합니다.

  시나리오 B: 결제 세션 생성 (/billing/checkout)
   1. Polar.sh 대시보드에서 테스트 상품(Product)을 하나 생성합니다.
   2. 생성된 상품의 Price ID를 가져옵니다.
   3. API 호출:
       * Endpoint: POST /api/v1/billing/checkout
       * Body: {"price_id": "polar_price_...", "success_url": "http://localhost:3000/success"}
       * 결과: 반환된 url을 브라우저에 입력했을 때 Polar 결제 창이 뜨는지 확인합니다.

  시나리오 C: 웹훅 테스트 (중요)
  실제 결제가 완료되었을 때 우리 DB에 반영되려면 Polar가 우리 서버로 웹훅을 보내야 합니다. 로컬
  환경에서는 ngrok 같은 도구가 필요합니다.

   1. ngrok 실행: ngrok http 8000
   2. Polar 설정: Polar 웹훅 설정에 ngrok 주소를 등록합니다. (예:
      https://abcd-123.ngrok-free.app/api/v1/webhooks/polar)
   3. 결제 테스트: Polar 결제창에서 테스트 카드로 결제 완료 후, 우리 DB의 subscriptions 테이블에
      데이터가 들어오는지 확인합니다.

  시나리오 D: 데스크톱 암호화 라이선스 검증
  작성해둔 client_example/secure_client.py를 활용합니다.

   1. secure_client.py 파일 내의 api_url, project_id, aes_key를 실제 값으로 수정합니다.
   2. 스크립트 실행: python secure_client.py
   3. 결과: 실제 패킷이 암호화되어 서버로 가고, 서버가 실제 DB를 조회하여 다시 암호화된 응답을 주는지
      확인합니다.

  4. 주의사항
   * Sandbox 모드: Polar.sh는 Sandbox(테스트) 환경을 지원하므로, 실제 돈이 나가지 않도록 반드시
     Sandbox 모드에서 테스트하세요.
   * JWT Secret: Supabase의 JWT Secret이 일치하지 않으면 401 Unauthorized 에러가 발생합니다.

  직접 테스트해보시고, 특정 단계(예: ngrok 설정이나 Polar API 호출)에서 막히는 부분이 있으면 바로
  말씀해 주세요! 도와드리겠습니다.
